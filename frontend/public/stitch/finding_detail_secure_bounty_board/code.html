Broken Access Control in Financial Reporting API

Reported by Specter_Dev • 14 hours ago

description Vulnerability Analysis

An IDOR (Insecure Direct Object Reference) vulnerability was discovered in the endpoint /api/v1/reports/export. An authenticated user can manipulate the report_id parameter to download proprietary financial statements of other organizations without proper authorization.

The backend fails to validate if the requested report_id belongs to the org_id associated with the requester's JWT token session. This allows for total horizontal privilege escalation across the ledger system.
